# 用代理让 Codex 接入 DeepSeek / Kimi / GLM（实战可用）

> 本文档不含任何 API Key，所有密钥位置都用 `<占位符>` 表示。**切勿把填了真实 key 的配置提交到 git。**

## 为什么必须加代理

Codex 官方**支持**自定义 provider（`[model_providers.<id>]`），但它只说一种协议——
**Responses API**（`wire_api` 只能是 `responses`，旧的 `chat` 已被官方移除）。

而 DeepSeek / Kimi(Moonshot) / 智谱 GLM 都只提供 **Chat Completions**。两边对不上口，
实测 Codex 直连 DeepSeek 会 `404 /v1/responses` 然后 `turn.failed`。

**解法**：本地跑一个翻译代理，对 Codex 暴露 Responses 端点、对上游说 Chat Completions。
本文用社区开源项目 **[CLIProxyAPI](https://github.com/router-for-me/CLIProxyAPI)**（第三方，非官方）。

```
Codex(responses) ──▶ CLIProxyAPI(127.0.0.1:8317) ──▶ DeepSeek / Kimi / GLM(chat)
```

## 一、下载代理（macOS arm64，免装 Go）

```bash
mkdir -p ~/cliproxyapi && cd ~/cliproxyapi
curl -sL -o cli.tar.gz \
  https://github.com/router-for-me/CLIProxyAPI/releases/latest/download/CLIProxyAPI_<版本>_darwin_aarch64.tar.gz
tar -xzf cli.tar.gz       # 得到 cli-proxy-api 可执行文件
```
（Intel Mac 用 `darwin_amd64`，Linux 用 `linux_amd64`，Windows 用 `windows_amd64`。最新版本号见 releases 页。）

## 二、写代理配置 `~/cliproxyapi/config.yaml`

```yaml
host: "127.0.0.1"          # 只监听本机
port: 8317
auth-dir: "~/.cli-proxy-api"
api-keys:
  - "<本地访问key-自己随便起一串>"   # Codex 用它连代理（不是上游真实 key）
openai-compatibility:
  - name: "deepseek"
    base-url: "https://api.deepseek.com/v1"
    api-key-entries:
      - api-key: "<你的 DeepSeek key>"
    models:
      - { name: "deepseek-v4-pro",   alias: "deepseek-v4-pro" }
      - { name: "deepseek-v4-flash", alias: "deepseek-v4-flash" }
  - name: "kimi"
    base-url: "https://api.moonshot.cn/v1"
    api-key-entries:
      - api-key: "<你的 Moonshot/Kimi key>"
    models:
      - { name: "kimi-k2.7-code", alias: "kimi-k2.7-code" }
      - { name: "kimi-k2.6",      alias: "kimi-k2.6" }
      - { name: "kimi-k2.5",      alias: "kimi-k2.5" }
  - name: "glm"
    base-url: "https://open.bigmodel.cn/api/paas/v4"
    api-key-entries:
      - api-key: "<你的 智谱 GLM key>"
    models:
      - { name: "glm-4.6", alias: "glm-4.6" }
      - { name: "glm-4.7", alias: "glm-4.7" }
      - { name: "glm-5",   alias: "glm-5" }
```
```bash
chmod 600 ~/cliproxyapi/config.yaml      # 收紧权限，里面是明文 key
```

代理按**模型名**路由：Codex 请求 `kimi-k2.7-code` 就会被转到 Moonshot，请求 `glm-4.6` 转到智谱，以此类推。所以三家共用同一个代理地址和同一个本地访问 key。

## 三、让代理常驻（开机自启 + 崩溃重启）

`~/Library/LaunchAgents/com.cliproxyapi.codex.plist`：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0"><dict>
  <key>Label</key><string>com.cliproxyapi.codex</string>
  <key>ProgramArguments</key>
  <array>
    <string>/Users/<你>/cliproxyapi/cli-proxy-api</string>
    <string>-config</string><string>/Users/<你>/cliproxyapi/config.yaml</string>
    <string>-local-model</string><string>-no-browser</string>
  </array>
  <key>WorkingDirectory</key><string>/Users/<你>/cliproxyapi</string>
  <key>RunAtLoad</key><true/>
  <key>KeepAlive</key><true/>
  <key>StandardOutPath</key><string>/Users/<你>/cliproxyapi/proxy.log</string>
  <key>StandardErrorPath</key><string>/Users/<你>/cliproxyapi/proxy.log</string>
</dict></plist>
```
```bash
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.cliproxyapi.codex.plist
launchctl enable   gui/$(id -u)/com.cliproxyapi.codex
curl -s http://127.0.0.1:8317/v1/models -H "Authorization: Bearer <本地访问key>"   # 应列出模型
```

## 四、把 Codex / 切换器指向代理

每个 provider 的块都指向代理（用模型切换器编辑即可，或手改 `~/.codex/config.toml`）：

```toml
[model_providers.deepseek]
name = "DeepSeek (经代理)"
base_url = "http://127.0.0.1:8317/v1"
wire_api = "responses"
experimental_bearer_token = "<本地访问key>"   # 注意：填代理的本地 key，不是 DeepSeek 真实 key
```
Kimi / GLM 同理，只是 `model` 用各自的模型名。**真实上游 key 只存在代理的 config.yaml 里**，不进 `~/.codex/config.toml`。

## 五、验证

```bash
# 用 Codex 非交互实跑一次（应返回模型回复，而不是 404）
CODEX_HOME=$(mktemp -d) /Applications/Codex.app/Contents/Resources/codex exec \
  -s read-only --skip-git-repo-check "Reply with: OK" </dev/null
```
桌面版里则需**开一个新会话**才生效。

## 管理命令

```bash
launchctl print     gui/$(id -u)/com.cliproxyapi.codex | grep state   # 状态
launchctl kickstart -k gui/$(id -u)/com.cliproxyapi.codex             # 重启（改完 config 后）
launchctl bootout   gui/$(id -u)/com.cliproxyapi.codex                # 停止并取消自启
tail -f ~/cliproxyapi/proxy.log                                       # 日志
```

## 说明

- CLIProxyAPI 是**第三方社区开源**项目，不是 OpenAI 官方；它经手你的上游真实 key（仅存本机 `config.yaml`，600 权限，只监听 127.0.0.1）。自行评估信任度。
- `Model metadata for ... not found` 是无害警告，不影响使用。
- 想再加别的 OpenAI 兼容模型：在 `openai-compatibility` 下照葫芦画瓢加一段上游 + 重启代理即可。
